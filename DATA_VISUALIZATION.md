# 数据可视化特性说明

本网站专门设计用于展示数据工程能力,包含多种数据可视化和呈现方式。

## 核心数据可视化组件

### 1. 技能雷达图 (SkillRadar)

**位置**: 首页 "技能可视化" 区域

**技术实现**: Recharts 雷达图

**数据展示**:
- 6 维技能评估(Python、SQL、数据仓库、AI工程、工程化、BI可视化)
- 0-100 刻度可视化
- 交互式 Tooltip 显示具体数值
- 渐变填充 + 半透明效果

**数据配置**: `src/content/skills.ts` 中的 `radarData`

```typescript
export const radarData = [
  { skill: 'Python', value: 90, fullMark: 100 },
  { skill: 'SQL', value: 95, fullMark: 100 },
  // ...
];
```

**可定制性**:
- 修改 `value` 调整技能水平
- 添加/删除维度
- 修改颜色(在 SkillRadar.tsx 中)

---

### 2. 项目指标卡片 (DataCard)

**位置**:
- 首页项目卡片中(每个项目显示 2 个指标)
- 项目详情页(显示全部 4 个指标)

**数据类型**:
- 对比型数据(前后对比)
- 绝对值数据
- 趋势指示(上升/下降/中性)

**示例**:
```typescript
<DataCard
  title="数据产出"
  after="10万条"
  improvement="质量90%+"
/>

<DataCard
  title="效率提升"
  before="4-6小时"
  after="15-30分钟"
  trend="down"  // 时间减少是好事
  improvement="80%"
/>
```

**视觉特性**:
- 大字号显示关键数值
- 前后对比用箭头连接
- 趋势图标(↑↓→)
- 改进百分比带颜色标识

---

## 数据驱动设计原则

### 1. 数字优先

所有关键成果用**量化数据**呈现:
- ✅ "效率提升 80%" > ❌ "效率显著提升"
- ✅ "SLA 99%+" > ❌ "高可用性"
- ✅ "10万条数据" > ❌ "大量数据"

### 2. 对比呈现

使用前后对比突出改进:
```
4-6小时 → 15-30分钟
60-70% → 90%+
数亿条记录,准确率99.99%
```

### 3. 视觉层次

- 大数字: 30-40px(核心指标)
- 中数字: 18-24px(次要指标)
- 小文字: 14-16px(说明文字)

### 4. 颜色语义

- 蓝色渐变(`#3b82f6 → #06b6d4`): 品牌色,技能/项目
- 绿色(`#10b981`): 正向成果(提升、增长)
- 红色(`#ef4444`): 负向成果(降低、减少,但实际是好的)
- 黄色(`#f59e0b`): 警示/注意

## 如何添加新的数据可视化

### 方式 1: 使用现有组件

```tsx
import DataCard from '@/components/ui/DataCard';

<DataCard
  title="新指标"
  before="旧值"
  after="新值"
  trend="up"
  improvement="提升50%"
/>
```

### 方式 2: 集成其他图表

安装额外的图表库:
```bash
npm install @tremor/react  # Tremor 图表库
```

在组件中使用:
```tsx
import { BarChart } from '@tremor/react';

<BarChart
  data={dataArray}
  index="name"
  categories={["value"]}
  colors={["blue"]}
/>
```

### 方式 3: 创建自定义可视化

1. 在 `src/components/` 创建新组件
2. 使用 Recharts / D3.js / Canvas
3. 在页面中导入使用

## 推荐的数据可视化扩展

### 短期扩展(1-2周)

1. **进度条图表**: 显示项目完成度
   ```tsx
   <ProgressBar value={85} label="数据质量" />
   ```

2. **对比柱状图**: 多项目指标对比
   ```tsx
   <BarChart data={projectComparison} />
   ```

3. **趋势折线图**: 时间序列数据(如质量提升曲线)
   ```tsx
   <LineChart data={qualityTrend} />
   ```

### 中期扩展(1个月)

1. **交互式数据看板**:
   - 可筛选的项目指标
   - 动态计算汇总数据
   - 导出数据功能

2. **技能矩阵图**:
   - 技能 × 项目矩阵
   - 热力图显示技能使用频率

3. **工作成果统计**:
   - 饼图: 项目类型分布
   - 漏斗图: 数据处理流程

### 长期扩展(3个月+)

1. **动态数据源**:
   - 连接 GitHub API 获取代码统计
   - 集成 LeetCode 刷题数据

2. **3D 可视化**:
   - Three.js 技能树
   - 项目关系网络图

3. **动画数据展示**:
   - 数字滚动动画
   - 图表渐入动画
   - 数据更新过渡动画

## 数据更新最佳实践

### 定期更新

- 每月更新技能雷达图数值
- 每季度更新项目指标
- 每半年更新工作经历

### A/B 测试

尝试不同的数据呈现方式:
- 雷达图 vs 条形图
- 数字卡片 vs 进度条
- 时间轴 vs 表格

### 性能优化

- 大数据集使用虚拟滚动
- 图表按需加载(懒加载)
- 使用 React.memo 避免重复渲染

## 数据源管理

所有数据集中在 `src/content/` 目录:

```
src/content/
├── personal.ts      # 个人信息 + 核心优势
├── skills.ts        # 技能 + 雷达图数据
├── experience.ts    # 工作经历
└── projects.ts      # 项目 + 指标数据
```

**优势**:
- 数据与视图分离
- 易于维护和更新
- 可复用数据
- 类型安全(TypeScript)

---

## 总结

本网站的数据可视化设计遵循以下原则:

1. **量化优先**: 所有成果用数字说话
2. **对比呈现**: 前后对比突出改进
3. **交互友好**: 悬停/点击显示详情
4. **响应式**: 移动端自适应布局
5. **性能优化**: 按需加载,避免卡顿
6. **可扩展**: 易于添加新图表类型

这些特性共同构成了一个以**数据为中心**的个人简历网站,完美展示数据工程能力。
